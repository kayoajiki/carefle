# ユーザージャーニー：オンボーディングフロー

## オンボーディングフェーズ（初回利用〜7日間）

### ステップ1: 現職満足度診断

#### 初回ログイン時
- **ユーザーの行動**: 初回ログイン時、自動で診断画面（`diagnosis.start`）にリダイレクトされる
- **体験**: 
  - 診断画面で質問に答えていく（約15-20分）
  - 診断は途中保存可能で、いつでもダッシュボードに戻れる
  - ダッシュボードに戻った場合、控えめなモーダル/オーバーレイで「診断を続けますか？」と促される
  - 「診断を続ける」ボタンと「後でやる」ボタンを表示
  - 一度「後でやる」を選択したら、しばらく表示しない（しつこくしない）
- **フィードバック**: 診断完了時に「診断が完了しました！次は今日の振り返りを書いてみましょう」と表示
- **次のアクション**: 「今日の振り返りを書く」ボタンが表示される

### ステップ2: 初回日記記録
- **ユーザーの行動**: 診断が終了しているユーザーには、ダッシュボード上にオーバーレイ/モーダルで「今日の振り返りを書く」が表示される
- **体験**: 
  - 通常の日記記録フォームで日記を書く（診断結果を参照したテーマ提案は不要）
  - 日記保存後、既存のAIフィードバックが表示される
- **フィードバック**: 日記保存後、既存のAIフィードバックが表示される（特別な「めちゃ褒める」機能は追加しない）
- **次のアクション**: 「自己診断結果を入力すると、もっと深い分析ができます」と案内

### ステップ3: 自己診断結果入力
- **ユーザーの行動**: ダッシュボードで「自己診断結果を1つ入力するとプチ取説が生成されます」と表示
- **体験**: MBTI、ストレングスファインダーなどの診断結果を入力（既に受けた診断結果を記録）
- **フィードバック**: 入力完了時に「診断結果が記録されました。あと7日間の日記でプチ取説が生成されます」と表示
- **次のアクション**: 「WCMシートを作成すると、未来の自分が見えてきます」と案内

### ステップ4: WCMシート作成
- **ユーザーの行動**: ダッシュボードで「WCMシートを作成する」をクリック
- **体験**: Will（やりたいこと）5問、Can（できること）5問、Must（やるべきこと）5問に答える
- **フィードバック**: WCMシート完成時に「未来の自分が描けました！次は7日間の日記を書いてみましょう」と表示
- **次のアクション**: 7日間の日記記録を促す

### ステップ5: 7日間の日記記録
- **ユーザーの行動**: 毎日、日記を書く（リマインド通知が送られる）
- **体験**: 
  - 日記を書くたびにAIが「めちゃ褒める」フィードバック
  - ダッシュボードに進捗バーが表示（例: 「あと3日でプチ取説が生成されます！」）
  - 連続記録日数が表示される（例: 「🔥 連続記録 3日」）
- **フィードバック**: 
  - 3日目: 「3日連続記録達成！素晴らしい習慣が身についてきました」
  - 7日目: 「7日間の記録が完了しました！プチ取説を生成しますか？」
- **次のアクション**: プチ取説生成ボタンをクリック

### ステップ6: プチ取説生成・閲覧
- **ユーザーの行動**: 「プチ取説を生成する」をクリック
- **体験**: 
  - 生成中はローディング表示（約30秒-1分）
  - 生成完了後、美しいレイアウトでプチ取説が表示される
  - 内容: 本質のみ（統合診断結果の核心、日記から見える価値観の本質、強みの核心、次のステップ提案）
- **フィードバック**: 「あなたのプチ取説が完成しました！」と表示
- **次のアクション**: 
  - SNSシェアボタン（X、Instagram、Facebook）
  - PDFダウンロード
  - 「もっと詳しく知りたい方は、日記を続けてみましょう」と案内

## 継続利用フェーズ（オンボーディング完了後）

### 日常的な利用
- **ユーザーの行動**: 毎日、または定期的に日記を書く
- **体験**: 
  - 日記を書くたびにAIが褒めるフィードバック
  - 連続記録日数が増えていく
  - ダッシュボードで進捗が可視化される
- **フィードバック**: 
  - 14日目: 「2週間連続記録達成！素晴らしい習慣です」
  - 30日目: 「1ヶ月連続記録達成！あなたの成長が感じられます」

### コンテキスト別取説の生成（段階的）
- **ユーザーの行動**: 特定のコンテキスト（仕事、家族など）について日記を書き続ける
- **体験**: 
  - 仕事関連の日記が5件以上蓄積されると、「仕事の自分」取説が生成可能になる
  - 家族関連の日記が5件以上蓄積されると、「家族の自分」取説が生成可能になる
- **フィードバック**: 「仕事の自分取説が生成できました！見てみますか？」と通知
- **次のアクション**: コンテキスト別取説を閲覧、SNSシェア

### 曼荼羅形式のマッピング生成
- **ユーザーの行動**: 十分な記録が蓄積された後、マッピング画面にアクセス
- **体験**: 
  - 曼荼羅形式で過去→現在→未来が可視化される
  - **外側の輪**：未来の自分（WCMのWill/Can/Must、マイルストーン）
  - **中央の輪**：現在の自分（最新の診断結果、最近の日記）
  - **内側の輪**：過去の自分（過去の診断、過去の日記、人生史）
  - 各コンテキスト（仕事、家族など）をセクションとして配置
  - クリックで詳細表示
- **フィードバック**: 「あなたの過去→現在→未来のマップが完成しました」
- **次のアクション**: 変容の可視化、過去の自分を思い出す

### 自分を思い出す機能
- **ユーザーの行動**: ダッシュボードの「過去の自分を思い出す」セクションにアクセス
- **体験**: 
  - 過去の日記、診断結果、取説へのクイックアクセス
  - 過去→現在→未来の変容を時系列で表示
  - 変容の大きさを視覚的に表示
- **フィードバック**: 「○ヶ月前の自分と比べて、こんなに成長しましたね」と表示
- **次のアクション**: 過去の記録を振り返り、現在の自分と比較

## 長期的な利用（3ヶ月以上）

### 変容の追跡
- **ユーザーの行動**: 定期的にマッピング画面を確認
- **体験**: 
  - 過去の自分と現在の自分を比較
  - 変容ポイントがハイライトされる
  - 成長グラフが表示される
- **フィードバック**: 「3ヶ月前と比べて、○○な面が強くなりましたね」と表示

### コンテキスト別取説の拡張
- **ユーザーの行動**: 複数のコンテキストで日記を書き続ける
- **体験**: 
  - 「仕事の自分」「家族の自分」「趣味の自分」など、複数の取説が生成される
  - 各取説が曼荼羅形式で統合される
- **フィードバック**: 「あなたの取説マップが完成しました」

### 将来的な拡張（ナノバナナ統合）
- **ユーザーの行動**: 曼荼羅形式の取説を画像としてエクスポート
- **体験**: 
  - 取説を画像として生成
  - カスタマイズ可能なスタイル
  - SNSシェア用の画像生成
- **フィードバック**: 「あなたの取説画像が完成しました」
