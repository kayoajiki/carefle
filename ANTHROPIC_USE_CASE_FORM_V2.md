# Anthropic使用目的フォーム提出ガイド（2024年12月版）

## 重要な変更点

**2024年12月時点で、AWS Bedrockの「モデルアクセス」ページは廃止されました。**

サーバーレスモデルは初回呼び出し時に自動的に有効化されるようになりました。ただし、Anthropicモデルについては、初回ユーザーは使用目的の詳細を提出する必要がある場合があります。

## 使用目的フォームを提出する方法

### 方法1: モデルカタログからプレイグラウンドで開く（推奨）

1. **AWS Bedrockコンソールにアクセス**
   - https://console.aws.amazon.com/bedrock/ にアクセス

2. **モデルカタログを開く**
   - 左メニューから「モデルカタログ」をクリック
   - または直接 https://console.aws.amazon.com/bedrock/home?region=us-east-1#/model-catalog にアクセス

3. **Anthropicモデルを選択**
   - モデル一覧から「Anthropic」のモデルを探す
   - 例: 「Claude 3 Haiku」または「Claude 3 Sonnet」
   - モデル名をクリック

4. **プレイグラウンドで開く**
   - モデル詳細ページで「プレイグラウンドで開く」ボタンをクリック
   - または「Chat」タブを選択

5. **使用目的フォームが表示される場合**
   - 初回使用時、使用目的フォームが自動的に表示されます
   - フォームを記入して提出してください

### 方法2: APIを直接呼び出す

1. **APIを呼び出す**
   - アプリケーションからモデルを呼び出すと、エラーレスポンスに使用目的フォームのリンクが含まれる場合があります

2. **エラーメッセージを確認**
   - エラーメッセージに使用目的フォームへのリンクが含まれている場合は、そのリンクからアクセス

### 方法3: AWS Marketplaceから有効化

1. **AWS Marketplaceにアクセス**
   - https://aws.amazon.com/marketplace/ にアクセス
   - 「Anthropic Claude」で検索

2. **モデルをサブスクライブ**
   - Anthropic Claudeモデルのページを開く
   - 「サブスクライブ」または「有効化」ボタンをクリック
   - 使用目的フォームが表示される場合は記入

## フォームの記入内容

- **使用目的**: 「General purpose / Chatbot」（一般的な用途/チャットボット）を選択
- **使用内容**: 「内省支援アプリケーションでの対話型チャット機能」など、具体的な用途を記入

## 承認までの時間

- 通常は数分で承認されます
- 最大15分程度かかる場合があります
- 承認後、モデルが使用可能になります

## 確認方法

1. **プレイグラウンドでテスト**
   - Bedrockコンソールのプレイグラウンドでモデルを呼び出してみる
   - 正常に応答が返ってくれば、有効化されています

2. **APIを呼び出してテスト**
   - アプリケーションからAPIを呼び出してみる
   - エラーが返らなければ、有効化されています

## トラブルシューティング

### フォームが表示されない

- モデルカタログから直接プレイグラウンドを開いてみる
- 別のブラウザで試す
- AWSアカウントの管理者権限があるか確認

### 承認が遅い

- 最大15分待ってください
- それでも承認されない場合は、AWSサポートに問い合わせ

### それでもエラーが出る

- `.env`ファイルの設定を確認
- アクセスキーが正しいか確認
- IAMユーザーに`AmazonBedrockFullAccess`ポリシーがアタッチされているか確認
- リージョンが正しいか確認（us-east-1推奨）
